## Directory Purposes

- Purpose: Contains GitHub Actions workflow definitions for CI/CD.
- Contains: YAML files defining automated tasks.
- Key files: `workflows/` (assumed, based on typical structure).

**`.planning/`:**
- Purpose: Stores planning and analysis documents generated by the AI agent.
- Contains: Markdown files.
- Key files: `codebase/ARCHITECTURE.md`, `codebase/STRUCTURE.md`, etc.

**`api/`:**
- Purpose: Houses the Python backend API service.
- Contains: Python source files, Dockerfile, dependency list.
- Key files: `main.py`, `requirements.txt`.

**`data-sync/`:**
- Purpose: Contains the Python data synchronization service.
- Contains: Python source files, Dockerfile, dependency list.
- Key files: `main.py`, `requirements.txt`.

**`frontend/`:**
- Purpose: Contains the React-based single-page application.
- Contains: TypeScript/TSX source files, CSS, static assets, configuration.
- Key files: `package.json`, `tsconfig.json`, `src/main.tsx`, `src/App.tsx`.

**`frontend/src/`:**
- Purpose: Core source code for the frontend application.
- Contains: React components, pages, hooks, API client.
- Key files: `App.tsx`, `main.tsx`, `index.css`.

**`frontend/src/api/`:**
- Purpose: Defines the client-side API interaction logic.
- Contains: TypeScript files with functions for making API requests and defining data types.
- Key files: `client.ts` (assumed, based on imports).

**`frontend/src/components/`:**
- Purpose: Reusable and modular UI components for the frontend.
- Contains: React components (TSX files).
- Key files: `ShipMarker.tsx`, `Map.tsx`, `SpeedChart.tsx`, `ShipList.tsx`, `ShipPanel.tsx`.

**`frontend/src/hooks/`:**
- Purpose: Custom React hooks for encapsulating reusable logic (e.g., data fetching).

**`frontend/src/pages/`:**

## Naming Conventions

**Files:**
- **Frontend:** PascalCase for React components and pages (e.g., `App.tsx`, `LiveMap.tsx`). camelCase for hooks (e.g., `useShips.ts`). kebab-case for CSS files (e.g., `index.css`).
- **Backend (Python):** snake_case for Python modules (e.g., `main.py`).

**Directories:**
- **Frontend:** kebab-case (e.g., `components`, `hooks`, `pages`).
- **Backend:** kebab-case (e.g., `api`, `data-sync`).

## Where to Add New Code

**New Feature (Frontend):**
- Primary code (new page): `frontend/src/pages/NewFeatureName.tsx`
- Primary code (new component): `frontend/src/components/NewComponentName.tsx`
- Primary code (new hook): `frontend/src/hooks/useNewHookName.ts`
- API client integration: `frontend/src/api/client.ts` (add new functions/types).
- Tests: `frontend/__tests__/` or co-located (not detected yet, needs convention).

**New API Endpoint (API Service):**
- Implementation: `api/main.py` (add new route handler).
- Dependencies: `api/requirements.txt` (if new libraries are needed).

**New Data Sync Logic (Data Sync Service):**
- Implementation: `data-sync/main.py` (add new processing logic).
- Dependencies: `data-sync/requirements.txt` (if new libraries are needed).

**New Component/Module:**
- **Frontend:** Follow existing patterns in `frontend/src/components/`, `frontend/src/pages/`, or `frontend/src/hooks/`.
- **Backend:** Create new Python modules within `api/` or `data-sync/` as needed.

**Utilities:**
- Shared helpers (frontend): `frontend/src/utils/` (create if needed).

## Special Directories

**`node_modules/` (frontend):**
- Purpose: Stores third-party JavaScript dependencies.
- Generated: Yes, by `npm install` or `yarn install`.
- Committed: No, excluded by `.gitignore`.

**`__pycache__/` (api/data-sync):**
- Purpose: Stores Python bytecode cache files.
- Generated: Yes, by Python interpreter.
- Committed: No, excluded by `.gitignore` (assumed).
